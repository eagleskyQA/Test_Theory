Уровни тестирования. "Позитивность" тестов.

Классификация тестирования по уровням, на которых оно проводится.

Уровень 1: компонентное, модульное, unit-тестирование;
Уровень 2: интеграционное тестирование;
Уровень 3: системное тестирование;

На компонентном уровне происходит тестирование отдельных модулей (к примеру в ИМ это тестирование корзины или оплаты заказа); на интеграционном происходит проверка этих модулей между собой; на системном полная проверка приложения.
Юнит-тестированием обычно занимается разработчик кода.

Под интеграционным тестированием понимают тестирование части системы состоящей из двух и более модулей. Например как со страницы корзины произвести оплату.

Существует две разновидности интеграционного тестирования:
1. Тестирование интеграций компонентов, т.е. то, как отдельные модули одного приложения взаимодействуют между собой;
2. Системное интеграционное тестирование, т.е. тестирование взаимодействия между всеми компонентами системы, либо взаимодействие разных систем междй собой, либо тестирование интерфейсов с помощью которых взаимодействует система.

Виды интерфейсов (API, CLI, GUI)
1. Интерфейс программирования приложений (API) - набор методов, которые можно использовать для доступа к функциональности других программ. Например для внедрения платежной системы для каого либо ресурса. Разработчик налаживает интеграцию с этой системой посредством данного интерфейса и ему не нужно разрабатывать свою платежную систему;
2)Интерфейс командной строки (CLI);
3)GUI - в нем программные функции представляются графическими элементами экрана. Например то, что мы видим, открывая страницу в интернете.

Системное тестирование выполняется на полной интегрированной системе с целью проверки системы исходным требованиям. Например в ИМ будем проверять, как все модули, которые были разработаны дляданного ИМ будут взаимодействовать между собой, можем ли мы пройтись по всем функциональностям нашего ИМ (зарегистрироваться, найти товар, положить его в корзину, оплатить и т.д.)

Так же существует приемочное тестирование. Это вид тестирования, проводимый на этапе сдачи готового продукта, либо его какой-то готовой части заказчику. Цель: определение готовности продукта. Достигается это путем прохода тестировочных сценариев, которые построены на основе спецификации требований к нашему продукту. Это финальный этап тестирования перед релизом. Обычно тестируется основной функционал, который не требует тщательных проверок.

Виды приемочного тестирования:
1) Пользовательское приемочное тестирование (UAT) User Acceptance Testing - проводится пользователем конечного продукта.
2) Эксплутационное - выполняется либо пользователями, либо администратором в среде, которая имитирует реальные условия рабочего окружения. Проводится проверка тестирования резервного копирования, аварийного восстановления системы. безопасность.
3) На соответствие контракту - проверка на соответствие спецификации, другому документу либо нормативному акту.
4) Альфа - эксплутационное тестирование со стороны разработчиков.
5) Бета - проводится с внешней стороны без участия разработчиков.

По позитивности тестирование бывает:
1) Позитивное - это тестирование, когда применяются сценарии, которые соответствуют нормальному, ожидаемому поведению системы (К примеру ввод верных данных);
2) негативное - это тестирование, в рамках которого применяются сценарии, которые соответствуют нештатному поведению системы (к примеру ввод неверных данных и реакции системы на это).
В первую очередь выполняются позитивные тесты!
